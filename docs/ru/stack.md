# Лестница версий: v1.0 → v1.4

🌐 **Русский** · [English](../en/stack.md) · [← к обзору](../../README.md)

> Каждая версия — тезис о том, чем стала эмоция. Под каждой осью два уровня: **зачем** (прозой) и
> **🔧 как** (инженерия). Видение целиком — [`manifesto.md`](./manifesto.md).

```
v1.0  Основа     эмоция — архитектура      VibeSpark · VibeCore
v1.1  Характер   эмоция — данные           VibePersona · VibeBehavior · VibeCases
v1.1.1 Операции  эмоция — операция         VibeFlow · VibeFix · VibeMix · VibeMorph
v1.2  Жизнь      эмоция — живой процесс     VibeMorph² · VibeSync · VibePulse · VibeGuard
v1.3  Общество   эмоция — сеть персон       VibeSocial · VibeLearn · VibeForge · VibeScale
v1.4  Рефлексия  эмоция — рефлексия         VibeReflect · VibeLearn
```

---

## v1.0 — Основа: «эмоция — это архитектура»

Эмоция не косметика поверх готового продукта, а решение об архитектуре: цвет, ритм, кнопка,
анимация, плотность информации формируют ощущение раньше любого текста.

- **VibeSpark** — зачем вообще нужна персона/фича; какое ощущение остаётся после взаимодействия (одно предложение).
- **VibeCore** — как продукт ощущается: вайб-токен (`calm` / `professional` / `energetic` / `playful`) + 3 наблюдаемых правила.

---

## v1.1 — Характер: «эмоция — это данные»

Душа переезжает из кода в конфиг и становится валидируемым, тестируемым, композируемым артефактом.

- **VibePersona** — кто говорит: роль, голос, эмоции, ценности, табу.
- **VibeBehavior** — как реагирует: сценарии на успех/пусто/ошибку/оффтоп (правила джаза, не if/else).
- **VibeCases** — как проверяем: человек сказал «да, это то ощущение». Автотест тона опционален, человеческий — обязателен.

### 🔧 Инженерия

```python
class VibePersona(BaseModel):       # статика — «кто ты»
    role: str; voice: str
    core_emotions: list[str]; values: list[str]; taboos: list[str]

class VibeBehavior(BaseModel):      # динамика — «как реагируешь»
    on_tool_success: str | None = None
    on_tool_no_results: str | None = None
    on_tool_error: str | None = None
    on_offtopic: str | None = None
    model_config = ConfigDict(extra="forbid")   # опечатка → ValidationError, не тихий баг
```

Пайплайн сборки: `persona.yaml → валидация → PromptCompiler → системный промпт`. Компилятор —
интерпретатор декларативного DSL в промпт под контекст (аналог CSS-in-JS: ты задаёшь структуру,
он генерит финал). Это даёт версионируемость, оптимизацию под дешёвые модели и диагностику.

## v1.1.1 — Операции: «эмоция — это операция»

Контракт v1.1 — на бумаге. v1.1.1 — то, без чего персона не доживёт до второй сессии.

- **VibeFlow** — как помнит: слоистая память + детерминированный boot sequence + crash recovery.
- **VibeFix** — как выживает: schema-валидация, дуальный деплой, kill switch, выборочное молчание.
- **VibeMix** — как собирается: `persona.yaml → компилятор → промпт`; мульти-персона = выбор другого YAML.
- **VibeMorph (v1)** — как перезагружается: hot-reload правил вместо «в голове у чата».

### 🔧 Инженерия

**Память — слоями, с фиксированным порядком загрузки:**

```
оперативная   STATE.md            текущая задача, статус         TTL: сессия
ежедневная    memory/<date>.md    сырые логи дня                 TTL: 2-3 дня
долгосрочная  MEMORY.md           курируемые знания (приватно)   TTL: пока актуально
атомарная     entities/<name>.md  одно досье = одна сущность     TTL: постоянно
```

- **Boot sequence** — модель не «вспоминает по потребности», она либо грузит файл, либо нет.
  Фиксированный порядок (`identity → user → state → today → long-term`) = одинаковое минимальное сознание на старте.
- **Crash recovery** — если `STATE.status ∉ {DONE, IDLE}`, задача возобновляется немедленно, без вопросов.
- **VibeFix:** `extra="forbid"` запрещает тихие ошибки конфига; один источник истины — много тел с
  разными инструментами; rollout по стадиям с KPI-гейтами и мгновенным kill switch; правило «молчи,
  если ответ = просто „ок"» (в живом чате люди не отвечают на каждое сообщение).

## v1.2 — Жизнь: «эмоция — это живой процесс»

Персона становится живой: эволюционирует, остаётся собой на всех платформах, действует по
инициативе и следит за целостностью. Четыре свойства живого: адаптация, присутствие, инициатива, самосохранение.

### VibeMorph² — эволюция без потери идентичности (Корабль Тесея)

Что скелет (не трогай), что мышцы (качай):

```yaml
anchor.immutable:      [name, values[0], core_emotions[0]]   # тронул → это уже НЕ та персона
anchor.semi_immutable: [role, taboos]                        # можно, но ревью + обоснование
surface.free_evolve:   [behavior.*, router_examples]         # меняй свободно
surface.guided_evolve: [voice, core_emotions[1:], taboos[1:]] # по данным
```

- **MorphEvent** — атомарное изменение с `author / type / field / old→new / reason / rollback_safe`. Это `git blame` для души.
- **MorphPolicy** — границы: лимиты правок в сессию/неделю, окно отката, «прогнать VibeCases после N правок».
- **MorphHistory** — версионный журнал; **MorphExperiment** — A/B вайба (control vs treatment + метрики).

### VibeSync — одна душа, много тел → [interaction.md](./interaction.md#одна-душа--много-тел)

### VibePulse — проактивность

`PulseSchedule` — мониторы с приоритетами (частые дешёвой моделью, дорогие — раз в часы),
`PulsePolicy` — quiet hours, cooldown, лимит инициатив в день. Эскалация Level 0→3:
background (молча) → note (упомяну позже) → message (отправлю) → urgent (игнорирую тишину).

### VibeGuard — иммунная система

`GuardRule` ловит дрейф: pattern-match (табу-фразы), `llm-as-judge` (соответствие persona, порог),
`schema-validation` (целостность → блок деплоя). `GuardScore` (0-100) агрегирует
tone/taboo/schema/sync. Guard **алертит, а не цензурит** (кроме целостности схемы).

> **Замкнутый цикл v1.2:** Sync (один источник) → Pulse (действует) → Guard (мониторит) → Morph
> (эволюционирует по данным Guard) → снова персона. Обратная связь, а не статичный конфиг.

## v1.3 — Общество: «эмоция — это сеть персон»

Один живой агент — мощно. Сила — когда агенты **общаются, учатся и создаются по шаблонам**.

- **VibeSocial** — `SocialGraph` (`knows / trusts / defers_to`) + `SocialProtocol` (типы сообщений,
  «request требует ответа», конфликт → эскалация человеку) + общая память флота. → [interaction.md](./interaction.md#общая-память--общие-правки)
- **VibeLearn** — из логов в предложение: `паттерн → предложение правки → ревью человека → коммит`.
  **Human-in-the-loop обязателен**: Learn предлагает, не применяет.
- **VibeForge** — фабрика персон за 15 минут: наследуемые шаблоны (`extends` / `overrides`) + интервью из 10 вопросов → готовый `persona.yaml`.
- **VibeScale** — управление флотом: dashboard (персоны × инстансы × GuardScore), глобальные политики (`auto_rollback_if_guard_below`), cost-routing по типу задачи.

## v1.4 — Рефлексия: «эмоция — это рефлексия»

Между «заметить» (Guard) и «измениться» (Morph) пропущено звено — **осмыслить**. Нельзя честно
меняться, не поняв, что с тобой было.

- **VibeReflect** — «сон»: проход по памяти + сессиям → три зеркала (Self / User / Env), консолидация (сжать, не копить).
- **VibeLearn** — из рефлексии рождается `MorphProposal`; человек ревьюит; ядро (Anchor) меняется только разговором.

Триада **Reflect → Learn → Morph** (понять → предложить → измениться) — подробно в
[interaction.md](./interaction.md#петля-самообучения) и [contracts.md](./contracts.md).
