# Контракты

🌐 **Русский** · [English](../en/contracts.md) · [← к обзору](../../README.md)

> Формальные схемы за словами «душа», «эволюция», «рефлексия». Каждая — данные с валидацией, статусом
> и (где надо) человеком на гейте. Машиночитаемые версии — в [`../../schemas/`](../../schemas).

---

## 1. Persona — душа как данные

```yaml
name: "<идентификатор>"
vibe:                       # статика: кто ты
  role: "..."
  voice: "..."
  core_emotions: [...]
  values: [...]
  taboos: [...]
behavior:                   # динамика: правила джаза (сценарии, не if/else)
  on_tool_success: "..."
  on_tool_no_results: "..."
  on_tool_error: "..."
  on_offtopic: "..."
tools: [ { name, type, description, router_examples } ]
```

Валидируется Pydantic с `extra="forbid"`: опечатка в имени поля = ошибка загрузки, а не тихий
неверный бот. `PromptCompiler` собирает из этого системный промпт под контекст выполнения.
Схема: [`persona.schema.json`](../../schemas/persona.schema.json).

## 2. Anchor / Surface — контракт эволюции

Идентичность держит **якорь**. Не всё в персоне можно менять одинаково.

| | Anchor (ядро / скелет) | Surface (поверхность / мышцы) |
|---|---|---|
| **Что** | имя, корневая ценность, доминантная эмоция; (для агента-разработчика — язык, базовые принципы, «не коммить секреты») | поведение, тон, вторичные эмоции/табу, форматы, интеграции |
| **Тронуть = ** | создать **другую** персону (смерть прежней) | эволюция той же персоны |
| **Как менять** | **только через разговор** с человеком | предложение → ревью → коммит |
| **Маркер** | `touches_anchor: true` | `touches_anchor: false` |

Правило: рефлексия может предложить что угодно, но Anchor неприкосновенен без диалога. Это иммунитет
против «сломай себя одним промптом».

## 3. MorphEvent — атомарное изменение (`git blame` для души)

```yaml
morph_event:
  id: "morph-2026-02-19-001"
  timestamp: "2026-02-19T15:00:00+03:00"
  author: "human"          # human | agent | system
  type: "behavior"         # anchor | surface | behavior | tool | value | emotion | taboo
  field: "behavior.on_offtopic"
  old_value: "..."
  new_value: "..."
  reason: "почему это изменение нужно"
  rollback_safe: true
```

Без MorphEvent через полгода никто не знает, *кто и зачем* поменял персону. Журнал всех событий —
`MorphHistory`. Схема: [`morph-event.schema.json`](../../schemas/morph-event.schema.json).

## 4. MorphProposal — предложение правки (выход VibeLearn)

Гипотеза, а не команда: агент приносит аргументированный diff и ждёт.

```yaml
morph_proposal:
  id: "2026-06-14_001a"
  target: "<файл/секция контракта>"
  change: "человекочитаемое описание правки"
  evidence: ["session:…", "reflection_pass:…"]   # на чём основано
  touches_anchor: false                           # true → обязателен разговор
  status: pending                                 # pending | approved | rejected | modified
```

**Правило ревью:** применяет правку **только человек**. `touches_anchor: false` — approve/reject/modify
одним действием; `touches_anchor: true` — сначала разговор с человеком, клик недопустим.
Схема: [`morph-proposal.schema.json`](../../schemas/morph-proposal.schema.json).

## 5. ReflectionPass — один «сон»

```yaml
reflection_pass:
  id: "2026-06-14_001"
  window: "last_7d"          # окно памяти/сессий, которое осмыслялось
  mirrors:
    self: [...]              # как я работал
    user: [...]              # рабочая модель человека (не досье)
    env:  [...]              # карта среды
  consolidated: 3            # сколько эпизодов сжато (метрика: память растёт сублинейно)
  proposals: ["2026-06-14_001a"]
```

Инвариант: если `consolidated` не растёт, а память пухнет — рефлексия **сломана**. «Сон» обязан сжимать.

---

## 6. Инварианты петли (anti-goals)

Контракт держится, пока выполняются все четыре:

1. **Нет автономной самоправки.** Изменение контракта — только через человека; Anchor — только разговором.
2. **Нет досье.** User-зеркало — читаемая и стираемая человеком рабочая модель, не скрытый профиль.
3. **Рефлексия сжимает.** Метрика успеха — память не растёт линейно.
4. **Нет мистики без артефакта.** За «сном» и «душой» — файл, схема, ревью. Нет артефакта → это шум.
