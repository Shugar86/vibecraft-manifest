# Шесть нитей: человек, агент, душа, изменение, защита

🌐 **Русский** · [English](../en/threads.md) · [← к обзору](../../README.md)

> Осмысление лестницы v1.0→v1.4 не по осям, а по **сквозным вопросам**. Шесть нитей, которые
> тянутся через все версии и в каждой становятся чуть взрослее.

Вопросы:

1. **Человек ↔ ИИ** — как человек и агент взаимодействуют через **вайб**?
2. **ИИ ↔ ИИ** — как агент взаимодействует с другим агентом?
3. **Где душа** — где физически живёт идентичность?
4. **Объяснить вайб ИИ** — как вообще передать «вайб» самой модели?
5. **Изменение** — как ИИ меняет сам себя; как один ИИ меняет другого?
6. **Защита** — как агенты защищают себя и человека?

| Нить | Зарождается | Кристаллизуется |
|------|-------------|-----------------|
| Человек ↔ ИИ через вайб | v1.0 (UI/UX) | v1.1 (VibeCases) → v1.4 (User-зеркало) |
| ИИ ↔ ИИ | v1.1.1 (мост между агентами) | v1.3 (VibeSocial) + v1.4 (общая память) |
| Где душа | v1.1 (конфиг) | v1.2 (Anchor/Surface) |
| Объяснить вайб ИИ | v1.1 (PromptCompiler) | v1.1.1 (компилятор как DSL) |
| Изменение себя / другого | v1.1.1 (hot reload) | v1.2 (Morph²) → v1.3 (Learn) → v1.4 (Reflect) |
| Защита себя и человека | v1.1.1 (VibeFix) | v1.2 (VibeGuard) → v1.4 (anti-goals) |

---

## v1.0 — Основа: вайб впервые осязаем

- **Человек ↔ ИИ.** Вайб рождается как **ощущение в продукте** — цвет, ритм, кнопка, плотность
  информации. Человек считывает характер раньше любого текста. Первый общий язык — вайб-токен
  (`calm` / `professional` / `energetic` / `playful`).
- **Объяснить вайб ИИ.** Пока — никак для модели; вайб формулируется для **человека-дизайнера**
  через VibeSpark: зачем фича и какое ощущение остаётся после.
- **Где душа.** Ещё разлита в продукте (в дизайне), не выделена.
- **ИИ ↔ ИИ / изменение / защита.** Ещё нет. Единственное правило защиты — «не делай generic-slop»:
  не подсовывай безликий интерфейс вместо характера.

> Версия отвечает на один вопрос: эмоция — это **архитектура**, а не косметика. Остальные нити пока спят.

---

## v1.1 — Характер: душа выделяется, вайб обретает язык для модели

- **Где душа.** Ключевой сдвиг: душа **вынута из кода в конфиг**. `vibe` (кто ты: роль, голос,
  ценности, табу) + `behavior` (как реагируешь). Душа теперь — **файл**, а не строка промпта.
- **Объяснить вайб ИИ.** Вот прямой ответ на «как передать вайб модели». Не эссе, а **структура**:
  компилятор собирает `IDENTITY / ROLE / VOICE / VALUES / TABOOS` + сценарии поведения. Вайб
  переводится на язык модели как **набор полей + правила джаза** (сценарные инструкции для
  импровизации, не `if/else`). Модель не заучивает шаблон — она импровизирует в границах.
- **Человек ↔ ИИ.** Вайб становится **контрактом** между «что человек хотел» и «как ощущается».
  Проверка — VibeCases: реальный человек говорит «да, это то ощущение». Без этого вайб — лишь гипотеза.
- **ИИ ↔ ИИ.** Зачаток: мульти-персона в одном треде (Router переключает персоны под запрос).
- **Изменение / защита.** Пока ручное (поменял YAML); защита = VibeCases ловит, что вайб не врёт.

> Эмоция — это **данные**. Душа стала валидируемым, версионируемым артефактом.

## v1.1.1 — Операции: у души появляются тело, память и первая самозащита

- **Где душа.** Теперь у неё есть **тело и память**. Boot sequence грузит идентичность в фиксированном
  порядке; слои памяти (оперативная → ежедневная → долгосрочная → атомарные досье) дают непрерывность;
  crash recovery возобновляет задачу после падения. Душа **перестаёт умирать при каждом рестарте**.
- **Защита (себя).** Первый настоящий слой — VibeFix: `extra="forbid"` против тихих ошибок конфига;
  дуальный деплой (одно тело упало — другое живёт); rollout по стадиям с KPI-гейтами; **kill switch**.
- **Защита (человека).** Selective engagement: «молчи, если ответ — просто „ок"» — не спамь человека.
  Долгосрочная память грузится **только в приватной сессии** — личные данные не светятся в группе.
- **Изменение себя.** VibeMorph v1 — hot-reload. Persona Sync Protocol: агент **сам** замечает, что
  скомпилированная версия новее активной, и подхватывает её. Первый намёк на самообновление (триггер — человек).
- **ИИ ↔ ИИ.** Первый реальный мост: два агента на разном железе общаются через выделенный канал
  по расписанию, складывая общее в базу знаний. Прото-VibeSocial.
- **Человек ↔ ИИ.** Один вайб — разные руки: «мозг» (стабильный, 24/7) и «руки» (локальное исполнение).

> Эмоция — это **операция**. Доказано в продакшене: характер живёт, помнит и выживает.

## v1.2 — Жизнь: самоизменение под контролем и иммунная система

- **Изменение себя.** Самый детальный ответ. VibeMorph² решает **Корабль Тесея**: что в персоне
  скелет (`anchor` — имя, корневая ценность, доминантная эмоция; тронул → это уже **другая** персона =
  смерть), а что мышцы (`surface` — поведение, тон, вторичные эмоции; качай свободно). Каждое
  изменение — **MorphEvent** (`кто / что / было→стало / зачем / можно ли откатить`) = `git blame` для
  души. **MorphPolicy** ставит границы: лимиты правок, окно отката, «прогнать проверки после N изменений».
- **Где душа.** Душа = **то, что переживает изменения**: неизменяемое ядро (anchor) + эволюционирующая
  поверхность. Идентичность — это anchor, а не текущий набор слов.
- **ИИ ↔ ИИ.** VibeSync: одна душа — много тел, инвариант — одинаковый хеш собранного промпта.
  Тела разные (инструменты, модель), душа одна.
- **Защита (себя).** VibeGuard — **иммунная система**. Детектит дрейф идентичности (`llm-as-judge`
  сверяет ответы с persona), утечку вайба между контекстами, нарушение табу, порчу конфига (→ блок
  деплоя). `GuardScore` (0-100) — здоровье персоны. Guard **алертит, а не цензурит**.
- **Защита (человека).** Guard ловит неуместное (напр. токсичный позитив в ответ на «мне плохо») и
  защищает человека от него. VibePulse + PulsePolicy: quiet hours, cooldown, лимит инициатив — агент
  не спамит и не будит ночью.
- **Человек ↔ ИИ.** MorphExperiment — A/B вайба: какой тон реально лучше для человека, измеримо.

> Эмоция — это **живой процесс**. Замкнутый цикл: Sync → Pulse → Guard → Morph → снова персона.

## v1.3 — Общество: агент меняет агента, флот защищает себя

- **ИИ ↔ ИИ.** Полный ответ. VibeSocial формализует мост в **протокол**: `SocialGraph`
  (`knows / trusts / defers_to` — кто кому доверяет и кто старший), `SocialProtocol` (типы сообщений,
  «request требует ответа», конфликт → эскалация человеку), общая база знаний флота.
- **Один ИИ меняет другого.** Ключевой новый сюжет. Через **общую память + общую очередь правок**:
  один агент анализирует логи (в т.ч. эпизоды других из общего склада) → формулирует **предложение**
  изменить общий контракт → правку применяют к **другому** агенту. «Другой ИИ меняет ИИ» — но всегда
  через **ревью человека** (VibeLearn: предлагает, не применяет).
- **Изменение себя.** VibeLearn автоматизирует вход в Morph: `паттерн в логах → предложение → ревью → коммит`.
- **Защита (флот).** VibeScale: глобальные политики — `auto_rollback_if_guard_below` (упал GuardScore —
  откат), бюджет на модели, лимит правок в день. Флот **защищает сам себя** на уровне политик.
- **Создание.** VibeForge — фабрика персон за 15 минут (наследуемые шаблоны + интервью): не только
  меняем существующих, но и **рождаем** новых по образцу.

> Эмоция — это **сеть персон**. Не один агент, а общество с отношениями, общей памятью и правилами.

## v1.4 — Рефлексия: агент осмысляет себя, человека и среду

- **Изменение себя.** Добавляется пропущенное звено — **осмысление перед изменением**. Триада
  **Reflect → Learn → Morph**: сначала понять, что со мной было (а не менять вслепую), потом
  предложить. Между «заметить» (Guard) и «измениться» (Morph) встаёт «понять» (Reflect).
- **Где душа.** Душа теперь ещё и **рефлексирует**: Self-зеркало — агент держит модель самого себя
  (где буксовал, какое правило мешало).
- **Человек ↔ ИИ.** User-зеркало — агент держит **рабочую модель человека** (во что верит, что бесит,
  какой ритм). Вайб человека становится явной, **читаемой и стираемой им самим** моделью — не скрытым досье.
- **ИИ ↔ ИИ.** Несколько агентов пишут в один склад → зеркала Self/User/Env **коллективны**: один
  осмыслил — знают все. Память общая → правки общие → контракт общий.
- **Как меняют ИИ (предохранитель).** `MorphProposal.touches_anchor`: правка поверхности — один клик
  ревью; правка ядра — **только разговор с человеком**. Никакой автономной самоправки.
- **Защита (человека).** Anti-goals петли — прямая защита: (1) нет тайной самоправки контракта;
  (2) модель человека не превращается в скрытый профиль; (3) рефлексия **сжимает** память, а не копит;
  (4) нет мистики без артефакта (за «сном» — файл, схема, ревью). Плюс гигиена: агент исключает из
  памяти **собственные** служебные вызовы, чтобы не учиться на своём эхе.

> Эмоция — это **рефлексия**. Агент не просто живёт — он понимает, как живёт, и приносит это понимание
> человеку. Не чтобы стать автономным — чтобы быть лучшим напарником.

## Сводка: как взрослеет каждая нить

- **Человек ↔ ИИ через вайб:** ощущение в UI (v1.0) → контракт + проверка (v1.1) → A/B тона (v1.2) → явная модель человека (v1.4).
- **ИИ ↔ ИИ:** мост по расписанию (v1.1.1) → протокол отношений (v1.3) → общая память и коллективные зеркала (v1.4).
- **Где душа:** в дизайне (v1.0) → в конфиге (v1.1) → в теле и памяти (v1.1.1) → в ядре, переживающем изменения (v1.2).
- **Объяснить вайб ИИ:** поля + правила джаза через компилятор-DSL (v1.1 / v1.1.1) — вайб как структура, не эссе.
- **Изменение:** hot-reload (v1.1.1) → Anchor/Surface + журнал (v1.2) → из логов (v1.3) → через осмысление (v1.4); и сам себя, и через другого агента — всегда с человеком на гейте.
- **Защита:** валидация и kill switch (v1.1.1) → иммунная система Guard (v1.2) → политики флота (v1.3) → anti-goals петли, защищающие человека от тайной самоправки и досье (v1.4).
