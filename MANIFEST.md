# Манифест VibeCraft v1.2

## 1. Философия: Эмоция как Архитектура

**VibeCraft** — это методология создания цифровых продуктов и AI-агентов, в котором эмоциональное восприятие («вайб») является не косметическим слоем, а фундаментальной частью архитектуры. Мы не «добавляем» настроение в конце — мы проектируем систему вокруг него.

**VibeCrafter** — это создатель (дизайнер, разработчик, продакт-менеджер, prompt-инженер), который лепит функциональность и ощущение как единое целое.

**VibePersona** — это AI-агент, обладающий характером, ценностями и уникальным стилем общения, а не просто текстовая модель, генерирующая ответы.

## 2. Принципы

*   **Цель — работающее настроение:** Конечный продукт — не просто работающее приложение, а работающее настроение, которое решает задачу пользователя.
*   **ИИ как инструмент-усилитель:** Искусственный интеллект — это наш молоток, стамеска и кисть. Он ускорает рутину, позволяя VibeCrafter'у сосредоточиться на видении и ощущениях.
*   **Первичен опыт, вторичны детали:** Мы проектируем от первого экрана к первому ощущению, и только потом — к детальной реализации.
*   **Эволюция через микрошаги:** Вайб продукта фиксируется и укрепляется через короткие, осязаемые итерации (`FlowCraft`).
*   **Характер — это данные, не код:** Всё, что определяет личность AI-агента, описывается в YAML/JSON. Ни одна черта характера не должна быть захардкожена в код.
*   **Живая персона эволюционирует:** AI-агент может меняться, но его ядро (Identity Anchor) остаётся неизменным. Корабль Тесея с рулём.

## 3. Полный Стек VibeCraft

### v1.0 — Основа
*   **VibeSpark**: Начальная искра идеи, объединяющая функцию и настроение. *Кто? Зачем? Как должно ощущаться?*
*   **VibeCore**: Минимальное ядро приложения с базовым стилем и одной ключевой функцией. Доказывает жизнеспособность «вайба».

### v1.1 — Характер (AI Personas)
*   **VibePersona**: Структурированное описание души агента. YAML-профиль с полями: `role`, `voice`, `core_emotions`, `values`, `taboos`. Не абстрактный промпт, а машиночитаемый контракт.
*   **VibeBehavior**: Динамика взаимодействия. Как агент реагирует на успех, ошибку, оффтоп, приветствие. Сценарий для импровизации, не жёсткий скрипт.
*   **VibeCases**: Живые тесты. Реальные сценарии, по которым проверяем, что персона ведёт себя как задумано.

### v1.1.1 — Операции
*   **VibeFlow**: Память и контекст. Thread_id, context extractor, anaphora rewriter. Как агент помнит разговор.
*   **VibeFix**: Устойчивость к шуму. NER, fuzzy matching, синонимы. Как агент выживает в реальных данных.
*   **VibeMix**: Микширование персон и скиллов. Композиция нескольких VibePersona в одного гибридного агента.
*   **PromptCompiler**: Трансляция persona.yaml → системный промпт. Данные, не ручная работа.

### v1.2 — Живая Персона
*   **VibeMorph**: Эволюция без потери идентичности.
  * *MorphAnchor* — неизменяемое ядро (name, values[0], core_emotions[0]). Изменил якорь — создал новую персону.
  * *MorphSurface* — эволюционируемая поверхность (behavior, voice, secondary emotions). Тренируй, настраивай.
  * *MorphEvent* — git blame для души. Каждое изменение с причиной и возможностью отката.
  * *MorphPolicy* — правила эволюции. Лимиты, rollback window, review requirements.
  * *MorphExperiment* — A/B-тестирование вайба с метриками.
*   **VibeSync**: Одна душа на всех платформах.
  * *SyncManifest* — что shared (persona.yaml, MEMORY.md), что per-instance (STATE.md, tools).
  * *SyncAdapter* — платформенная адаптация (Telegram vs VK vs API).
  * *SyncIdentity* — SHA-хеш persona.yaml совпадает на всех инстансах.
*   **VibePulse**: Проактивная агентность.
  * *PulseSchedule* — расписание мониторов с приоритетами (high/medium/low).
  * *PulseAction* — уровни действий: background (тихо) → note (потом) → message (отправить) → urgent (игнорируй тишину).
  * *PulsePolicy* — quiet hours, cooldown, max_daily_initiatives.
*   **VibeGuard**: Иммунная система персоны.
  * *GuardRule* — правила мониторинга (паттерны taboo violations, identity drift).
  * *GuardAudit* — периодическая самопроверка (daily/weekly/monthly).
  * *GuardScore* — композитная метрика здоровья персоны (0-100).

## 4. Отличие от "Vibe Coding"

| Аспект | VibeCraft | Vibe Coding |
| :--- | :--- | :--- |
| **Приоритет** | Настроение → Функция | Скорость → Функция |
| **Роль ИИ** | Инструмент для лепки ощущений + персона с характером | Генератор кода по запросу |
| **Результат** | Продукт с уникальным характером и живым агентом | Быстро созданный прототип |
| **Масштаб** | От UI до AI-персон и мультиагентных систем | Одиночные задачи |

## 5. VibePersona — Структура

```yaml
name: "agent_name"

vibe:
  role: "Кто это? Функциональная роль."
  voice: "Как говорит? Стиль речи."
  core_emotions: ["dominant_emotion", "secondary_emotion"]
  values: ["Что защищаем любой ценой"]
  taboos: ["Чего никогда не делаем"]

behavior:
  on_success: "Как отвечает при успехе"
  on_error: "Как отвечает при ошибке"
  on_greeting: "Как здоровается"
  on_offtopic: "Как реагирует на оффтоп"
  routing_style: "strict | helpful"

pulse:
  frequency: "10m | 30m | 1h"
  priority: "high | medium | low"
  executor: "cron | heartbeat"

guard:
  identity_anchor: ["name", "vibe.values[0]"]
  max_morph_per_week: 3
```

## 6. Production Реализация

VibeCraft проверен в продакшене:
*   **VibePack v4.0** — строгая tenant isolation через ResponseComposer
*   **PromptCompiler** — YAML → системный промпт
*   **Pydantic валидация** с `extra="forbid"` — нет тихих ошибок
*   **15 минут** на добавление нового агента (только YAML + тесты)
